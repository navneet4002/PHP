Loops in PHP

A loop is a cyclic repetition of a process that executes a specific task. The condition is checked:

  • If the condition is true, the loop iterates.
  • The loop keeps iterating until the condition becomes false.
  • When false, the loop stops and control is sent outside the loop.

Types of Loops in PHP

  1. while loop
  2. do…while loop
  3. for loop
  4. foreach loop

While Loop Syntax

initialize the loop_variable;
while(condition) {
    // Code
    // Increment / Decrement the loop_variable
}

Notes: loop_variable is a regular variable used to create the loop condition. Initialize it with a start value.

Do-While Loop Syntax

initialize the loop_variable;
do {
    // Code
    // Increment / Decrement the loop_variable
} while(condition);

For Loop Syntax

for(initialization; condition; increment/decrement) {
    // Code
}

Arrays

→ An array is a special variable.

→ An array can store multiple values in a single variable.

→ Three types of arrays:

  • Indexed array
  • Associative array
  • Multidimensional array

Indexed Array

→ Syntax for empty indexed array declaration:

$array_name = array();

→ Syntax to declare an indexed array with values:

$array_name = array(value1, value2, value3);

→ Every array element has a unique index: min index = 0 & max index = size – 1

→ Access an array element using: $array_name[index]

→ Get the size of the array using: count($array_name)

→ Display the whole array using: print_r($array_name)

→ Access all array elements in a loop:

for($index_var = 0; $index_var < count($array_name); $index_var++) {
    echo $array_name[$index_var];
}

Indexed Array Example 1

<?php
// Declare an indexed array with values
$cities = array("New Delhi", "Mumbai", "Chennai", "Kolkatta");

// Get the array size
echo("<br>Total Cities: " . count($cities));
echo("<br>");

// Display the whole array
print_r($cities);

// Access individual array elements
echo("<br>City at index 0: " . $cities[0]);
echo("<br>City at index 2: " . $cities[2]);

// Access all elements using a loop
echo("<ul>");
for($i = 0; $i < count($cities); $i++) {
    echo("<li>".$cities[$i]."</li>");
}
echo("</ul>");
?>

The __construct function

→ Special function called automatically when an object is created.

  • Name fixed: __construct()
  • May / may not take arguments
  • Cannot return a value
  • Only one constructor allowed per class
  • Can pass arguments to set data while creating object

Inheritance in PHP

→ Inheritance is creating a new class and deriving properties from a previously created class.

Benefits:

  • Reusability of code
  • Reduces code size, effort, time, and complexity

Types of Inheritance:

  1. Single Level
  2. Multi-Level
  3. Hierarchical

Single Level Inheritance

class SuperClass { }
class SubClass extends SuperClass { }

Notes:

  • Private members of SuperClass cannot be accessed in SubClass
  • Protected members accessible in SubClass but not outside
  • Public members accessible in SubClass

Multi-Level Inheritance

class SuperClass { }
class IntermediateSuperClass extends SuperClass { }
class SubClass extends IntermediateSuperClass { }

Hierarchical Inheritance

class SuperClass { }
class SubClass1 extends SuperClass { }
class SubClass2 extends SuperClass { }

Note: PHP does not support multiple inheritance directly:

class SubClass extends SuperClass1, SuperClass2 { } // This is NOT valid PHP

Example (MyProduct.php)

<?php
class Product {
    protected $id;
    protected $name;
    protected $brand;
    protected $price;

    public function setProduct($id, $name, $brand, $price) {
        $this->id = $id;
        $this->name = $name;
        $this->brand = $brand;
        $this->price = $price;
    }

    public function getId() { return $this->id; }
    public function getName() { return $this->name; }
    public function getBrand() { return $this->brand; }
    public function getPrice() { return $this->price; }
}
?>

PHP MySQL Database Connectivity

  • → We can store data received from the user to a MySQL database for permanent storage.
  • → PHP can connect with any RDBMS like MySQL, Microsoft SQL Server, Oracle DB, etc.
  • → Database Connectivity cases:
    • Executing a DML statement (INSERT, UPDATE, DELETE)
    • Executing a SELECT statement to fetch single/multiple rows
    • Executing a SELECT statement for record existence verification

Case 1: Executing a DML Statement (INSERT, UPDATE, DELETE)

<?php
// --- Configuration ---
$serverName = "localhost"; // Or your server IP/hostname
$dbUsername = "root";      // Your database username
$dbPassword = "your_password"; // Your database password (replace!)
$dbName = "phpdb";         // Your database name
$port = 3306;             // Default MySQL port (change if different)

// --- Establish Connection ---
$connection_object = mysqli_connect(
    $serverName,
    $dbUsername,
    $dbPassword,
    $dbName,
    $port
);

// --- Check Connection ---
if(mysqli_connect_errno()) { // Simplified check
    die("Database connection failed: " . mysqli_connect_error());
}

// --- Prepare Data (Example - Get from a form POST request) ---
// IMPORTANT: Sanitize user input before using in queries to prevent SQL injection!
$unm = isset($_POST['username']) ? mysqli_real_escape_string($connection_object, $_POST['username']) : '';
$pass = isset($_POST['password']) ? mysqli_real_escape_string($connection_object, $_POST['password']) : ''; // Hash passwords in real apps!
$fnm = isset($_POST['fullname']) ? mysqli_real_escape_string($connection_object, $_POST['fullname']) : '';
$gen = isset($_POST['gender']) ? mysqli_real_escape_string($connection_object, $_POST['gender']) : '';
$dob = isset($_POST['dob']) ? mysqli_real_escape_string($connection_object, $_POST['dob']) : ''; // Validate date format

// --- Build the Query ---
// Note: Using prepared statements is generally safer than mysqli_real_escape_string
$query = "INSERT INTO userinfo (username, password, fullname, gender, dob)
          VALUES('$unm','$pass','$fnm','$gen','$dob');";

// --- Execute the Query ---
if(mysqli_query($connection_object, $query) === false) {
    echo("Error executing query: " . mysqli_error($connection_object)); // Show error
} else {
    echo("Query executed successfully.");
    // --- Check Rows Affected (optional but good for confirmation) ---
    $rows_affected_var = mysqli_affected_rows($connection_object);
    echo (" Rows affected: " . $rows_affected_var);
}

// --- Close Connection ---
mysqli_close($connection_object);
?>

State Management

  • → By default, data of a page is not sharable across pages.
  • → State management helps share data across pages without forms.
  • → Technologies: Query String, Sessions, Cookies

Query String

  • Transports limited string data from source page to target page
  • Type & quantity: Strings up to ~2-4kb (browser dependent)
  • Storage: Client Browser's address bar (URL)
  • Security: Insecure (visible, easily manipulated)
  • Scope: Available only on the target page accessed via that specific URL
  • Syntax (single): TargetPageURL?key=value
  • Syntax (multiple): TargetPageURL?key1=value1&key2=value2
  • Retrieve in target page: $var = $_GET["key"]; (Check if set first!)

Sessions

  • → PHP Sessions store and pass information across multiple pages temporarily (during a user's visit).
  • → Widely used for user logins, shopping carts, temporary user preferences.
  • → Session data is stored on the **server**. A unique session ID (usually stored in a cookie) links the user's browser to their server-side data.
  • → Properties of Sessions:
    • **Type & Quantity:** Any PHP data type (strings, numbers, arrays, objects), virtually unlimited size (limited by server resources).
    • **Storage Location:** Server’s temporary storage (configured in `php.ini`).
    • **Security:** More secure than cookies or query strings as data isn't stored/visible in the browser. Session ID hijacking is a potential risk to mitigate.
    • **Scope:** Accessible across all pages within the same website/application during the session.
    • **Life:** Lasts until the session is explicitly destroyed (e.g., logout), the browser is closed (typically), or the server's session timeout is reached.

Sessions Syntax

  • **Start/Resume Session:** Must be called at the very top of **every** PHP file that needs access to session data, before any output. 
    session_start();
  • **Create/Assign Session Data:** Use the `$_SESSION` superglobal array. 
    $_SESSION["session_key"] = value; // e.g., $_SESSION["username"] = "john_doe";
  • **Retrieve Session Data:** Access the `$_SESSION` array. Always check if the key exists first. 
    if (isset($_SESSION["session_key"])) {
    $var = $_SESSION["session_key"];
    // Use $var
} else {
    // Session variable not set
}
  • **Destroy a Specific Session Variable:** 
    unset($_SESSION["session_key"]);
  • **Destroy the Entire Session:** Removes all session variables and ends the session. Often used for logout. 
    // 1. Unset all session variables
$_SESSION = array(); // Or session_unset(); prior to PHP 5.4.0

// 2. Destroy the session cookie (optional but recommended)
if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();
    setcookie(session_name(), '', time() - 42000,
        $params["path"], $params["domain"],
        $params["secure"], $params["httponly"]
    );
}

// 3. Finally, destroy the session data on the server
session_destroy();

Cookies

  • → Cookies store small pieces of information directly in the **user's browser**.
  • → Often used for "remember me" functionality, tracking user preferences (like themes), or analytics.
  • → The server sends the cookie to the browser, and the browser sends it back to the server with subsequent requests to the same domain.
  • → Properties of Cookies:
    • **Type & Quantity:** Primarily **strings**, limited size (around 4KB per cookie, and a limited number of cookies per domain, browser-dependent).
    • **Storage Location:** Client's browser storage.
    • **Security:** **Insecure** for sensitive data. Cookies are sent with every request and can be viewed/modified by the user or intercepted (if not using HTTPS). Use `HttpOnly` and `Secure` flags for mitigation.
    • **Scope:** Accessible across pages on the same domain (path can be restricted).
    • **Life:** Can be set to expire at a specific time/date or when the browser session ends.

Cookie Syntax

  • **Create/Set Cookie:** Must be called **before any HTML output**. 
    setcookie(name, value, expire, path, domain, secure, httponly);
    • `name`: The cookie name (string).
    • `value`: The cookie value (string).
    • `expire`: Expiration timestamp (integer). Use `time() + seconds`. 0 means expire at end of session. Negative means delete.
    • `path`: (Optional) Server path the cookie is available on ('/' for entire domain).
    • `domain`: (Optional) Domain the cookie is available to.
    • `secure`: (Optional) `true` if only sent over HTTPS.
    • `httponly`: (Optional) `true` if not accessible via JavaScript (helps prevent XSS).
    Example: Set for 1 hour: 
    setcookie("username", "john_doe", time() + 3600, "/", "", true, true); // Expires in 1 hour, HTTPS only, JS inaccessible
  • **Retrieve Cookie:** Use the `$_COOKIE` superglobal array. Always check if set. 
    if (isset($_COOKIE["cookie_name"])) {
    $value = $_COOKIE["cookie_name"];
    // Use $value (remember to sanitize if displaying)
} else {
    // Cookie not set
}
  • **Delete Cookie:** Set the expiration date to the past. 
    setcookie("cookie_name", "", time() - 3600); // Set value empty and expire in the past

Interfaces

  • → An interface defines a **contract** for classes. It specifies methods that a class **must** implement, but not how they are implemented.
  • → Defined using the interface keyword.
  • → Can only contain **public abstract method signatures** (no bodies) and **constants**. Cannot contain properties (variables).
  • → Cannot be instantiated directly (new InterfaceName() is not allowed).
  • → A class uses the implements keyword to adopt an interface. A class can implement multiple interfaces, separated by commas.
  • → The implementing class **must** provide a concrete implementation for **all** methods defined in the interface(s), using the exact same signature (name, parameters, type hints).
  • → Useful for enforcing a specific structure or capability across different, possibly unrelated, classes (achieving a form of polymorphism).

Interface Syntax

interface InterfaceName {
    // Constants (optional)
    const SOME_CONSTANT = 'value';

    // Method signatures (must be public)
    public function methodName(type $param1, $param2): ?returnType;
    public function anotherMethod();
}

class ClassName implements InterfaceName { //, AnotherInterface { // Can implement multiple
    // Constants can be accessed via InterfaceName::SOME_CONSTANT

    // Must implement all methods from InterfaceName
    public function methodName(type $param1, $param2): ?returnType {
        // Concrete implementation (method body) here
    }

    public function anotherMethod() {
        // Concrete implementation here
    }

    // Other class properties and methods
}

Example: IFace1

<?php
// Define the interface
interface IFace1 {
    public function fun1(); // Method signature, no body
}

// Implement the interface
class MyClass implements IFace1 {
    // Provide the concrete implementation for fun1
    public function fun1() {
        echo "<br/>MyClass implementation of fun1() runs.";
    }

    // Can have other methods too
    public function otherMethod() {
        echo "<br/>This is another method in MyClass.";
    }
}

Abstract Classes & Polymorphism

Abstract Methods & Classes

  • → An **abstract method** is declared in a parent class using the abstract keyword but contains no implementation (no body `{}`). It defines only the required signature. 
    abstract protected function calculatePay(); // No body
  • → An **abstract class** is a class that contains at least one abstract method. It must also be declared with the abstract keyword. 
    abstract class Employee { ... }
  • → Abstract classes **cannot be instantiated** directly (new Employee() is not allowed if `Employee` is abstract). They serve as blueprints for derived classes.
  • → Any **child class** that extends an abstract class **must** provide a concrete implementation for **all** inherited abstract methods, or the child class must also be declared abstract.
  • → Abstract classes can contain regular properties and concrete (non-abstract) methods, just like normal classes.
  • → Abstract methods can be `public` or `protected`, but not `private`.

Use abstract classes when you want to provide a common base structure and potentially some shared concrete implementation, while forcing subclasses to provide specific implementations for certain methods.

Example: Employee Abstract Class (Employee.php)

<?php
abstract class Employee {
    // Properties (can be public, protected, or private)
    protected $empid;
    protected $empname;
    protected $workinghours;
    protected $workingdays;

    // Concrete method (shared implementation)
    public function setEmpData($empid, $empname, $workinghours, $workingdays) {
        $this->empid = $empid;
        $this->empname = $empname;
        $this->workinghours = $workinghours;
        $this->workingdays = $workingdays;
    }

    // Concrete getter (example)
     public function getEmpName(): string {
         return $this->empname ?? 'N/A';
     }

    // Abstract method - MUST be implemented by child classes
    abstract public function calculateSalary(): float; // Added return type hint
}
?>

Derived Classes Implementing Abstract Method

<?php
require_once("Employee.php");

class PermanentEmployee extends Employee {
    private const BASE_BONUS = 10000;
    private const HOURLY_RATE = 100;

    // Concrete implementation of the abstract method
    public function calculateSalary(): float {
        // Ensure properties are set before calculation
        if (!isset($this->workinghours, $this->workingdays)) {
             return 0.0; // Or throw an exception
        }
        $basePay = $this->workinghours * self::HOURLY_RATE * $this->workingdays;
        return $basePay + self::BASE_BONUS;
    }
}

class TemporaryEmployee extends Employee {
     private const HOURLY_RATE = 50;

     // Concrete implementation of the abstract method
     public function calculateSalary(): float {
         if (!isset($this->workinghours, $this->workingdays)) {
             return 0.0;
         }
         return $this->workinghours * self::HOURLY_RATE * $this->workingdays;
     }
}
?>

Polymorphism

  • → **Poly** (many) + **Morph** (forms). The ability of an object or method to take on many forms.
  • → In OOP, it often means that different classes can respond to the same method call (defined in a parent class or interface) in their own specific way.
  • → Allows you to treat objects of different classes uniformly if they share a common interface or abstract parent class.
  • → Implemented in PHP primarily via:
    • **Abstract Classes:** Child classes provide different implementations for the same abstract method signature (as shown in the Employee example).
    • **Interfaces:** Different classes provide their own implementations for the methods defined in the interface contract (as shown in the Greeting example).
  • → Makes code more flexible, extensible, and easier to manage. You can work with a collection of `Employee` objects without needing to know if each one is `Permanent` or `Temporary` when calling `calculateSalary()`.

Polymorphism using Abstract Class (Recap)

In the Employee example above, calling $employee->calculateSalary() exhibits polymorphism. The exact calculation performed depends on whether $employee is an instance of PermanentEmployee or TemporaryEmployee, even though the method call looks the same.

Polymorphism using Interface (Example)

<?php
interface Greeting {
    public function greet(): string; // Added return type
}

class English implements Greeting {
    public function greet(): string { return "Hello!"; }
}

class German implements Greeting {
    public function greet(): string { return "Hallo!"; }
}

class French implements Greeting {
    public function greet(): string { return "Bonjour!"; }
}

// Function that accepts any object implementing the Greeting interface
function printGreeting(Greeting $person) {
    echo "<br/>Greeting: " . htmlspecialchars($person->greet());
}

// Usage
$person_eng = new English();
$person_ger = new German();
$person_fr = new French();

printGreeting($person_eng); // Outputs: Greeting: Hello!
printGreeting($person_ger); // Outputs: Greeting: Hallo!
printGreeting($person_fr); // Outputs: Greeting: Bonjour!

// We can treat them uniformly because they all implement Greeting
$people = [$person_eng, $person_ger, $person_fr];
foreach ($people as $p) {
     if ($p instanceof Greeting) { // Good practice to check
        printGreeting($p);
     }
}
?>
